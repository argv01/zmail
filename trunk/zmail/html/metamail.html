Metamail/mp Build Hints

Syd Logan

This page indicates which makefile is used to build metamail/mp for a given platform, and what changes are needed in order for the builds to occur. Perhaps we should build some host-specific Makefiles based on these changes (e.g. Makefile.hp700_10) so that these steps are not needed each time a clone is done. Note except for solaris, nothing needs to be done for mp.

Note that minor source changes were made to metamail sources that correspond to the CFLAGS changes described below. These changes have been checked into CVS, and need to be considered if we upgrade to a newer version of metamail sometime in the future.

  • m88k4
    cp Makefile.sysv Makefile
  • hp700
    cp Makefile.sysv Makefile, add -D_HPUX_SOURCE to CFLAGS
  • hp700_10
    cp Makefile.sysv Makefile, add -D_HPUX_SOURCE to CFLAGS
  • hp800_10
    cp Makefile.sysv Makefile, add -D_HPUX_SOURCE to CFLAGS
  • osf1v3
    cp Makefile.bsd Makefile
  • sun411
    cp Makefile.sysv Makefile
  • sun413
    cp Makefile.sysv Makefile
  • sun414
    cp Makefile.sysv Makefile
  • caldera
    cp Makefile.sysv Makefile, add -DLINUX to CFLAGS
  • sco
    cp Makefile.sysv Makefile, add -DSCO to CFLAGS
  • aix4
    cp Makefile.sysv Makefile, add -DAIX4
  • irix53
    cp Makefile.sysv Makefile
  • irix62
    cp Makefile.sysv Makefile, add -DIRIX62
  • sol23
    cp Makefile.sysv Makefile
    For mp, change its makefile to use gcc, and add -I/usr/openwin/include to CDEF macro
  • sol24
    cp Makefile.sysv Makefile
    For mp, change its makefile to use gcc, and add -I/usr/openwin/include to CDEF macro
  • sol25cc
    cp Makefile.sysv Makefile
    For mp, add -I/usr/openwin/include to CDEF macro in makefile
