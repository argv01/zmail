GNATS for the Emacs Impaired

You can file bug reports and append comments to bug reports without ever using emacs.

File Bug Reports

Use the gnats template, shown at the end of this page.

  1. Change the following fields in your template, and save the template in your $templates directory: 
    >Originator:  put your name here
>Release:     version of the software you are using
>Environment: for windows, the version of windows and your stack
>Category:    see /zen/usr/local/lib/gnats/z-code. 
	      qa requests:        qa-test
	      zmail for windows:  win16-port
>Class:       <[ sw-bug | doc-bug | change-request | support ]>
  2. When you use the template, do not fill in the Subject header. Instead, use the Synopsis field:
    >Synopsis: description containing as many keywords as possible
    Fill in the "Description" of your bug or test request, and if you know "How to Repeat" the behavior, include that information also.

Append Comments to a Bug Report

If you receive a bug report message and would like to add comments:

  1. Reply to the message. Do not alter the subject line.
  2. Cc: bugs

You can remove all the bug report fields from your reply -- makes the bug report easier to read.

You cannot change the state of a bug (open, analyzed, feedback, closed) through mail. For that, you need emacs-full GNATS or the edit-pr command.
